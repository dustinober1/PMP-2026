# 6.4 ‚Äì Resource and Procurement Planning

## 6.4.1 Introduction

Projects require resources‚Äîpeople, equipment, materials, and sometimes external vendors. This section covers planning for acquiring and managing these resources.

::: tip Key Insight
**Resource planning bridges scope and execution**. You can't execute without the right resources at the right time.
:::

---

## 6.4.2 Resource Planning

### Types of Resources

| Type            | Examples                               |
| --------------- | -------------------------------------- |
| **Human**       | Team members, contractors, consultants |
| **Physical**    | Equipment, materials, facilities       |
| **Financial**   | Budget, funding                        |
| **Information** | Data, documents, expertise             |

### Resource Requirements

For each activity, identify:
- **Type** of resource needed
- **Quantity** required
- **Skills/specifications**
- **When** needed (timing)
- **Duration** of need

---

## 6.4.3 Resource Estimation

### Approaches

| Approach                 | Description                           |
| ------------------------ | ------------------------------------- |
| **Expert judgment**      | Ask those who know the work           |
| **Alternative analysis** | Compare different resource options    |
| **Published data**       | Industry standards, benchmarks        |
| **Bottom-up**            | Estimate for each activity, aggregate |

### Considerations

- Skill level affects duration
- Availability affects scheduling
- Location affects cost
- Experience affects quality

## 6.4.5 Resource Optimization

### Resource Leveling

Adjusts the schedule to address resource constraints:

- May extend schedule to avoid overallocation
- Prioritizes critical path activities
- Smooths resource demand

**Trade-off**: Schedule may get longer

### Resource Smoothing

Adjusts activities within their float:

- Does not extend schedule
- Uses available slack
- May not resolve all conflicts

**Trade-off**: Limited improvement possible

## 6.4.6 Team Acquisition

### Pre-Assignment

Some team members assigned before planning:
- Named in charter or proposal
- Required for specialized expertise
- Committed by functional managers

### Negotiation

Working with functional managers:
- Clarify skills and timing needed
- Understand competing priorities
- Document agreements
- Handle conflicts constructively

### Acquisition Challenges

| Challenge              | Response                               |
| ---------------------- | -------------------------------------- |
| Key skills unavailable | Training, outsourcing, adjust schedule |
| Part-time allocation   | Account for context switching          |
| Virtual team           | Address communication, collaboration   |
| Competing projects     | Prioritize with leadership             |

## 6.4.7 Procurement Planning

### Make-or-Buy Analysis

Decide whether to:
- **Make**: Do work internally
- **Buy**: Hire external resources

### Factors in Make-or-Buy

| Factor          | Make (Internal)     | Buy (External)         |
| --------------- | ------------------- | ---------------------- |
| Core competency | Build capability    | Non-core, outsource    |
| Cost            | Cheaper internally  | Cheaper externally     |
| Capacity        | Resources available | Constrained internally |
| Control         | Need tight control  | Can accept external    |
| Speed           | Can meet timeline   | Faster externally      |
| Risk            | Manageable          | Transfer to vendor     |

## 6.4.8 Contract Types

### Fixed Price Contracts

| Type                                              | Description                  | Risk              |
| ------------------------------------------------- | ---------------------------- | ----------------- |
| **Firm Fixed Price (FFP)**                        | Set price, no adjustment     | Seller bears risk |
| **Fixed Price Incentive Fee (FPIF)**              | Base price + incentive       | Shared risk       |
| **Fixed Price Economic Price Adjustment (FPEPA)** | Adjusts for economic changes | Shared risk       |

**Best for**: Well-defined scope, stable requirements

### Cost-Reimbursable Contracts

| Type                               | Description             | Risk                  |
| ---------------------------------- | ----------------------- | --------------------- |
| **Cost Plus Fixed Fee (CPFF)**     | Cost + set fee          | Buyer bears risk      |
| **Cost Plus Incentive Fee (CPIF)** | Cost + incentive fee    | Shared risk           |
| **Cost Plus Award Fee (CPAF)**     | Cost + subjective award | Buyer bears most risk |

**Best for**: Undefined scope, R&D, uncertainty

### Time and Materials (T&M)

- Pay for hours worked + materials
- Moderate risk sharing
- Best for: Unknown duration, staff augmentation

::: warning Exam Alert
Know which contract type puts risk on buyer vs. seller:
- Fixed price ‚Üí Risk on **seller**
- Cost-reimbursable ‚Üí Risk on **buyer**
:::

## 6.4.9 Procurement Documents

### Common Documents

| Document                          | Purpose                                   |
| --------------------------------- | ----------------------------------------- |
| **RFI** (Request for Information) | Gather information from potential vendors |
| **RFP** (Request for Proposal)    | Solicit proposals (emphasis on approach)  |
| **RFQ** (Request for Quotation)   | Solicit price quotes (defined scope)      |
| **IFB** (Invitation for Bid)      | Competitive bidding (lowest price wins)   |

### Statement of Work (SOW)

Describes work to be performed:
- **Performance-based**: Outcomes required
- **Functional**: Capabilities needed
- **Design**: Detailed specifications

## 6.4.10 Vendor Selection

### Selection Criteria

| Criterion      | Considerations                       |
| -------------- | ------------------------------------ |
| **Price**      | Total cost, not just unit price      |
| **Quality**    | Track record, references             |
| **Experience** | Similar projects, industry knowledge |
| **Capacity**   | Ability to deliver on time           |
| **Approach**   | Technical solution, methodology      |
| **Risk**       | Financial stability, dependencies    |

### Source Selection Methods

| Method                   | Description                           |
| ------------------------ | ------------------------------------- |
| **Lowest price**         | Cheapest bid wins                     |
| **Best value**           | Balance price and other factors       |
| **Qualifications-based** | Select on capability, negotiate price |
| **Sole source**          | Only one viable vendor                |

---

## 6.4.11 Key Takeaways

1. **Plan resources systematically**: Type, quantity, skills, timing
2. **Optimize allocation**: Leveling may extend schedule; smoothing uses float
3. **Acquire through negotiation**: Work with functional managers
4. **Make-or-buy analysis**: Consider core competency, cost, control, risk
5. **Know contract types**: Fixed price (seller risk) vs. cost-plus (buyer risk)
6. **Select vendors carefully**: Price is not the only criterion
