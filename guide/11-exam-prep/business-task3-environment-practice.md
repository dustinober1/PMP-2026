# Business Task 3: Evaluate and Address External Business Environment Changes for Impact on Scope
## 50 Hard-Mode Practice Questions

**Domain**: Business Environment (26%)
**Task**: Evaluate and address external business environment changes for impact on scope
**Format**: Scenario-based single-best-answer - Select 1 correct answer from 4 options

---

## Instructions

Each question presents a scenario involving an external change (Political, Economic, Social, Technological, Legal, or Environmental) that impacts project scope. **Exactly 1 answer is correct.** The questions are designed to test your ability to evaluate, analyze, and pivot rather than just react.

::: tip Exam Strategy
For Business Environment questions, the FIRST step is almost always to **Evaluate** or **Assess** before you act. Look for "Trap" answers that skip right to a solution (e.g., "Update the project plan") without first understanding the impact on the business case or scope.
:::

### Question 1

**Topic:** Survey External Changes | **Methodology:** Predictive

**Scenario:**
You are managing a 2-year infrastructure project in a foreign country. Six months into execution, the local government announces a 25% increase in import tariffs on heavy machinery, which was not factored into the original budget or procurement plan. This change will significantly increase the cost of critical equipment needed for the final phase.

**Question:**
What is the FIRST action for the project manager to take?

A) Submit a formal change request to increase the project budget to cover the tariff costs
B) Negotiate with current vendors to see if they can absorb the tariff costs under existing contracts
C) Perform an impact analysis to determine how this change affects the project's bottom line and business case
D) Inform the project sponsor that the project may need to be cancelled due to unforeseen economic shifts

<details>
<summary>View Answer</summary>

**Correct Answer:** C

**Rationales:**
- **C (Correct):** According to PMP 2026 principles, when an external environment change occurs, the PM must first assess and prioritize the impact on project scope/backlog. You cannot recommend options (like budget increases) without a full analysis.
- **A (Incorrect):** This is a secondary step; you only submit a change request AFTER the impact analysis is completed and options are evaluated.
- **B (Incorrect):** While potentially part of a solution, you first need to know the magnitude of the impact on the overall project before negotiating piecemeal.
- **D (Incorrect):** This is an extreme reaction before any analysis has been performed to see if the project remains viable.

**Reference:** ECO Business Task 3; PMBOK 7 Section 2.5 (Project Environment)
</details>

### Question 2

**Topic:** Assess Scope Impact | **Methodology:** Agile

**Scenario:**
A team is developing a fintech application using Scrum. Mid-project, the central bank announces a surprise interest rate hike that shifts consumer preference from "Spending Rewards" to "Savings Incentives." The Product Owner wants to immediately stop work on the current "Cashback" module and start on a "High-Yield Savings" feature.

**Question:**
What is the BEST way to address this environmental change?

A) Immediately pivot the team and update the current Sprint Backlog to reflect the new priority
B) Conduct a backlog refinement session to assess the impact of the rate hike on the remaining stories and re-prioritize based on new value
C) Inform the Product Owner that the current Sprint must be completed before any changes can be considered
D) Ask the team to work overtime to finish the Cashback module while starting the High-Yield Savings feature

<details>
<summary>View Answer</summary>

**Correct Answer:** B

**Rationales:**
- **B (Correct):** In Agile, external market/economic shifts require collaborative backlog refinement. Assessing value and re-prioritizing ensures the team delivers what is currently most valuable to the business.
- **A (Incorrect):** "Immediately" changing the Sprint Backlog is a trap; it disrupts the team's commitment. Changes usually happen during refinement or the next planning session unless the Sprint is cancelled.
- **C (Incorrect):** This is too rigid; if the current work no longer has value due to an external shift, the PM/PO should evaluate if the Sprint Goal is still relevant.
- **D (Incorrect):** This leads to burnout and ignores the need to deprioritize low-value work.

## ðŸ’¡ Key Takeaways: Handling External Scope Changes

1.  **Assess Before Action**: Never jump to updating the project plan or budget without first performing an **impact analysis** on the existing scope and business case.
2.  **PESTLE Analysis**: Use the PESTLE framework (Political, Economic, Social, Technological, Legal, Environmental) to survey the environment for potential changes.
3.  **The Pivot**: PMP 2026 focuses heavily on the ability to **"Pivot"**. If an external change makes a feature irrelevant, find the next highest-value path rather than blindly following the original plan.
4.  **Agile Refinement**: In Agile environments, the **Product Backlog Refinement** and **Sprint Review** are your primary tools for evaluating and addressing environmental changes.
5.  **Alignment**: Project success is defined by **Business Value**, not just staying on schedule. If the environment changes such that your scope no longer provides value, you MUST recommend a pivot or cancellation.
