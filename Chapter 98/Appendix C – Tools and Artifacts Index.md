# Appendix C – Tools and Artifacts Index

This appendix provides a quick-reference index of the project management tools, techniques, and artifacts covered throughout the PMP 2026 Study Guide. It maps these items to the specific chapters where they are discussed in context.

## 1. Artifacts (Inputs & Outputs)

These are the documents, plans, and baselines that govern the project.

| Artifact | Description | Primary Coverage | Domain |
| :--- | :--- | :--- | :--- |
| **Assumption Log** | Records all assumptions and constraints throughout the project life cycle. | [Chapter 10b](../Chapter%2010/10b%20–%20Understanding%20Constraints,%20Assumptions,%20and%20Dependencies.md) | Process |
| **Backlog (Product)** | An ordered list of everything that is known to be needed in the product. | [Chapter 11b](../Chapter%2011/11b%20–%20Scope%20Definition%20and%20Baseline.md) | Process |
| **Business Case** | Documents the economic feasibility and strategic validity of the project. | [Chapter 10a](../Chapter%2010/10a%20–%20Business%20Case%20and%20Project%20Justification.md) | Business Env |
| **Change Log** | A comprehensive list of changes made during the project. | [Chapter 14c](../Chapter%2014/14c%20–%20Change%20Control%20and%20Configuration%20Management.md) | Process |
| **Issue Log** | A document where information about issues is recorded and monitored. | [Chapter 12d](../Chapter%2012/12d%20–%20Issue%20Management.md) | Process |
| **Lessons Learned Register** | Records knowledge gained during the project to improve future performance. | [Chapter 16f](../Chapter%2016/16f%20–%20Continuous%20Improvement%20and%20Organizational%20Learning.md) | Process |
| **Project Charter** | Formally authorizes the project and provides the project manager with authority. | [Chapter 10c](../Chapter%2010/10c%20–%20Developing%20the%20Project%20Charter.md) | Process |
| **Project Management Plan** | The document that describes how the project will be executed, monitored, and controlled. | [Chapter 11](../Chapter%2011/11%20–%20Planning%20for%20Value%20Delivery.md) | Process |
| **Risk Register** | A repository in which outputs of risk management processes are recorded. | [Chapter 12b](../Chapter%2012/12b%20–%20Risk%20Identification%20and%20Analysis.md) | Process |
| **Stakeholder Register** | Identifies the people, groups, or organizations that have an interest in the project. | [Chapter 7a](../Chapter%207/7a%20–%20Stakeholder%20Identification%20and%20Analysis.md) | People |
| **Work Breakdown Structure (WBS)** | A hierarchical decomposition of the total scope of work. | [Chapter 11b](../Chapter%2011/11b%20–%20Scope%20Definition%20and%20Baseline.md) | Process |

Techniques used to collect, evaluate, and understand project information.

| Tool / Technique | Description | Primary Coverage |
| :--- | :--- | :--- |
| **Alternatives Analysis** | Evaluating options to select the best approach. | [Chapter 8d](../Chapter%208/8d%20–%20Decision-Making%20Frameworks%20and%20Escalation.md) |
| **Benchmarking** | Comparing actual or planned practices to those of comparable organizations. | [Chapter 11e](../Chapter%2011/11e%20–%20Quality%20Planning%20and%20Acceptance%20Criteria.md) |
| **Brainstorming** | A technique used to identify a list of ideas in a short period of time. | [Chapter 12b](../Chapter%2012/12b%20–%20Risk%20Identification%20and%20Analysis.md) |
| **Cost-Benefit Analysis** | Financial analysis tool used to estimate the strengths and weaknesses of alternatives. | [Chapter 10a](../Chapter%2010/10a%20–%20Business%20Case%20and%20Project%20Justification.md) |
| **Earned Value Analysis (EVA)** | Compares the performance measurement baseline to the actual schedule and cost performance. | [Chapter 14a](../Chapter%2014/14a%20–%20Tracking%20Progress%20and%20Performance.md) |
| **Interviews** | Formal or informal approach to elicit information from stakeholders. | [Chapter 11a](../Chapter%2011/11a%20–%20Requirements%20Collection%20and%20Refinement.md) |
| **Root Cause Analysis** | An analytical technique used to determine the basic underlying reason that causes a variance or defect. | [Chapter 13b](../Chapter%2013/13b%20–%20Quality%20Assurance%20and%20Testing.md) |
| **SWOT Analysis** | Analysis of strengths, weaknesses, opportunities, and threats. | [Chapter 15a](../Chapter%2015/15a%20–%20Organizational%20Strategy%20and%20Project%20Selection.md) |
| **Variance Analysis** | Determining the cause and degree of difference between the baseline and actual performance. | [Chapter 14a](../Chapter%2014/14a%20–%20Tracking%20Progress%20and%20Performance.md) |

## 3. Decision Making & Estimating Tools

Techniques for making choices and forecasting.

| Tool / Technique | Description | Primary Coverage |
| :--- | :--- | :--- |
| **Analogous Estimating** | Using values from a previous, similar activity or project as the basis for estimating. | [Chapter 11d](../Chapter%2011/11d%20–%20Cost%20Estimation%20and%20Budgeting.md) |
| **Bottom-up Estimating** | Aggregating the estimates of the lower-level components of the WBS. | [Chapter 11d](../Chapter%2011/11d%20–%20Cost%20Estimation%20and%20Budgeting.md) |
| **Decision Tree Analysis** | A diagramming and calculation technique for evaluating the implications of a chain of multiple options. | [Chapter 12b](../Chapter%2012/12b%20–%20Risk%20Identification%20and%20Analysis.md) |
| **Multicriteria Decision Analysis** | Using a decision matrix to evaluate and rank ideas. | [Chapter 8d](../Chapter%208/8d%20–%20Decision-Making%20Frameworks%20and%20Escalation.md) |
| **Parametric Estimating** | Using a statistical relationship between historical data and other variables. | [Chapter 11d](../Chapter%2011/11d%20–%20Cost%20Estimation%20and%20Budgeting.md) |
| **Planning Poker** | An agile estimating technique that uses consensus to estimate effort. | [Chapter 11c](../Chapter%2011/11c%20–%20Schedule%20Planning%20and%20Work%20Breakdown.md) |
| **Three-Point Estimating** | Analyzing the optimistic, pessimistic, and most likely estimates. | [Chapter 11d](../Chapter%2011/11d%20–%20Cost%20Estimation%20and%20Budgeting.md) |
| **Voting** | A collective decision-making technique (e.g., unanimity, majority, plurality). | [Chapter 8d](../Chapter%208/8d%20–%20Decision-Making%20Frameworks%20and%20Escalation.md) |
